# 附录 A 常见术语表

本附录用于统一全书术语。第一次阅读时不必逐项背诵，遇到不熟悉的词可以回到这里查定义和工程含义。

| 术语 | 英文 | 简要解释 | 移动端工程关注点 |
| --- | --- | --- | --- |
| 大语言模型 | Large Language Model, LLM | 基于大量文本和代码训练的生成式模型，可以根据上下文生成文本、代码或结构化结果。 | 不要把模型当作确定性业务系统；关键动作必须有服务端校验。 |
| Token | Token | 模型处理文本的基本单位，可能是一个字、一个词或词的一部分。 | 影响上下文长度、费用、延迟和截断风险。 |
| 上下文窗口 | Context Window | 一次请求中模型能看到的最大 Token 范围。 | 聊天记录、检索片段和系统规则都要竞争窗口。 |
| Prompt | Prompt | 发送给模型的指令、上下文、约束和示例的组合。 | Prompt 模板应放在服务端版本化管理，客户端只传业务参数。 |
| 系统指令 | System Instruction | 约束模型角色、边界和输出行为的高优先级指令。 | 不应让用户输入覆盖系统指令。 |
| Few-shot 示例 | Few-shot Examples | 在 Prompt 中提供少量输入输出样例，引导模型按期望格式回答。 | 适合分类、抽取、格式化等稳定任务。 |
| 温度 | Temperature | 控制采样随机性的参数，越高越发散，越低越稳定。 | 故障分析、权限说明、JSON 输出通常使用低温度。 |
| Top-p | Nucleus Sampling | 从累计概率达到阈值的候选集合中采样。 | 不建议和 temperature 同时大幅调高。 |
| 幻觉 | Hallucination | 模型生成看似合理但缺乏依据或事实错误的内容。 | UI 应展示来源、置信度和“需要人工确认”的边界。 |
| Embedding | Embedding | 把文本映射为向量，用于计算语义相似度。 | 常用于知识库搜索、相似问题推荐和语义去重。 |
| 向量检索 | Vector Search | 根据向量距离或相似度找出相关文本片段。 | 需要处理权限、版本、排序和召回质量。 |
| RAG | Retrieval-Augmented Generation | 先检索外部资料，再把资料放入上下文让模型生成答案。 | 适合文档问答、故障手册、客服知识库和内部助手。 |
| Chunk | Chunk | 文档切分后的片段。 | 太短会缺上下文，太长会挤占窗口；需要结合标题和元数据。 |
| 重排 | Reranking | 对初步召回结果做二次排序。 | 能提升引用质量，但会增加延迟和成本。 |
| 结构化输出 | Structured Output | 要求模型按 JSON Schema 或固定字段返回结果。 | 不能只靠正则修补，服务端必须做 Schema 校验。 |
| 工具调用 | Tool Calling | 模型选择调用某个函数或业务工具，并生成参数。 | 模型只负责建议工具和参数，权限与写操作必须由服务端执行。 |
| Agent | Agent | 能在目标、工具、观察结果之间循环决策的模型驱动流程。 | 移动端更适合展示过程、确认高风险动作和处理超时取消。 |
| Workflow | Workflow | 由固定节点和明确条件组成的业务流程。 | 比 Agent 更可控，适合审批、周报、工单和通知。 |
| SSE | Server-Sent Events | 服务端通过长连接持续向客户端推送事件。 | 适合流式回答；移动端需要处理断线、取消、重试和完成态。 |
| P95 延迟 | P95 Latency | 95% 请求都能低于该值的延迟指标。 | 比平均值更能反映用户体感。 |
| Prompt Injection | Prompt Injection | 用户或外部资料试图覆盖系统规则、窃取上下文或诱导违规工具调用。 | 外部资料必须作为不可信上下文处理。 |
| 脱敏 | Redaction | 在日志、提示词或报告中隐藏邮箱、手机号、密钥等敏感值。 | 移动端日志和服务端日志都不应保存原始隐私数据。 |
| ATS | App Transport Security | Apple 平台的网络安全策略，默认鼓励 HTTPS。 | Debug HTTP 例外不能进入生产配置。 |
| Cleartext HTTP | Cleartext HTTP | 未加密的 HTTP 明文传输。 | Android 调试可配置例外，生产必须使用 HTTPS。 |
