# 前言

第一册讨论的是如何把大模型能力接入移动端应用；第二册讨论的是另一个更贴近日常研发的问题：当 AI coding agent 已经能够读仓库、改文件、运行命令、提交 PR 时，移动端工程师应该如何重新设计自己的开发方式。

移动端项目天然复杂。它有 UI 状态、平台差异、构建脚本、依赖管理、证书签名、设备兼容、性能指标、隐私合规、灰度发布和线上崩溃。把一个通用 AI 助手直接丢进这样的工程里，往往只能得到一次性的代码片段。要得到稳定收益，必须把 agent 放进正确的工程环境中。

本书把 AI coding 能力拆成四类：

- Prompt Engineering：把任务、边界和交付物说清楚。
- Context Engineering：让模型看到足够但不过量的仓库上下文。
- Harness Engineering：准备工具、权限、测试、脚本、沙箱和自动化护栏。
- Loop Engineering：设计理解、修改、验证、复盘的反馈循环。

这四类能力共同决定了 agent 能否完成真实任务。会写提示词只是开始，能让 agent 在复杂工程中稳定工作，才是 AI coding 专家的分水岭。
