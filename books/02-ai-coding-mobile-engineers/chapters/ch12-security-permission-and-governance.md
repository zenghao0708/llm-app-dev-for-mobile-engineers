# 第 12 章 权限、安全与团队治理

## 本章导读

AI coding agent 可以读文件、改代码、运行命令，意味着它必须被纳入安全治理。越强大的 agent，越需要明确权限、审计和回滚机制。

## 学习目标

- 知道 AI coding 中常见安全风险。
- 能够设计移动端仓库的权限分层。
- 掌握团队治理和代码审查要求。

## 风险清单

- 泄露 API Key、证书、token、日志和用户数据。
- 修改签名、发布、生产配置。
- 引入未审查依赖。
- 执行危险命令。
- 生成不符合隐私合规的权限说明。
- 在不了解业务规则的情况下修改支付、登录、风控代码。

## 权限分层

| 权限级别 | 允许动作 | 适用任务 |
| --- | --- | --- |
| 只读 | 搜索、阅读、解释 | 代码理解、方案评审 |
| 受限写 | 修改指定目录 | 小 bug、测试、文档 |
| 受限执行 | 运行白名单命令 | 测试和构建 |
| 人工确认 | 依赖、权限、签名、发布 | 高风险变更 |
| 禁止 | 密钥、生产证书、删除历史 | 不应交给 agent |

## 团队治理

团队需要定义：

- 哪些任务可以直接由 agent 做。
- 哪些任务必须开 PR。
- 哪些文件变更必须人工审查。
- 生成代码是否需要额外测试。
- agent 输出是否纳入审计。

## 本章小结

安全不是阻止使用 AI coding，而是给它建立边界。没有边界的自动化最终会变成不可控风险。
