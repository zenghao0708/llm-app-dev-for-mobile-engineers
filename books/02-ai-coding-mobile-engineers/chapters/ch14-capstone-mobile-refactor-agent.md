# 第 14 章 综合项目：移动端重构 Agent 工作流

## 本章导读

本章把前面的四类工程能力串起来，设计一个可复用的移动端重构 agent 工作流。目标不是展示某个工具的演示效果，而是形成一套团队可执行的流程。

## 项目目标

选择一个移动端项目中的真实问题：网络错误处理分散在多个页面，导致 UI 文案不一致、埋点不完整、测试覆盖不足。我们希望 agent 帮助完成一次受控重构。

## 工作流阶段

1. 任务定义：说明重构目标和禁止事项。
2. 上下文收集：读取网络层、错误模型、页面调用方和测试。
3. 计划生成：列出最小修改路径。
4. 第一轮修改：只抽取统一错误模型，不改 UI。
5. 验证：运行单元测试和静态检查。
6. 第二轮修改：迁移两个低风险页面。
7. 人工审查：确认文案、埋点和平台行为。
8. 扩展迁移：按页面批次推进。
9. 交付：生成 PR 描述、测试结果和风险说明。

## Prompt 模板

```text
你是移动端重构助手。任务是把网络错误处理迁移到统一错误模型。

约束：
- 不修改签名、发布、证书和生产开关。
- 不改变登录、支付和风控逻辑。
- 每轮最多迁移两个页面。
- 每轮必须运行指定测试。

输出：
- 本轮计划。
- 修改文件列表。
- 测试命令和结果。
- 仍需人工审查的点。
```

## 验收清单

- 统一错误模型有单元测试。
- 页面迁移有回归测试或快照验证。
- 文案进入本地化资源。
- 埋点字段不丢失。
- 弱网和超时有兜底。
- PR 描述包含迁移范围和剩余页面。

## 本章小结

综合项目的关键不是让 agent 一次做完所有重构，而是把大任务拆成可验证的小循环。人类负责边界、优先级和审查，agent 负责执行、反馈和重复性工作。
