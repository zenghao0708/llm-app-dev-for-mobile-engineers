# 第 10 章 调试、重构与测试自动化

## 本章导读

AI coding 最容易产生稳定收益的三类任务是调试、重构和测试补充。它们都有明确输入、明确目标和可验证结果。

## 学习目标

- 掌握用 agent 处理 bug、重构和测试的任务拆分方式。
- 知道如何控制重构范围。
- 能够让测试成为 agent 的反馈信号。

## 调试任务

调试任务要给 agent 足够现场：

- 崩溃堆栈。
- 设备型号和系统版本。
- 最近变更。
- 复现步骤。
- 日志片段。
- 期望行为。

Agent 不应该直接猜修复方案，而应该先建立假设，再用测试或日志验证。

## 重构任务

重构任务的关键是边界：

- 一次只重构一个模块或一种重复模式。
- 禁止同时改业务行为和架构。
- 先加测试或快照，再改实现。
- 输出迁移说明和回滚方式。

## 测试自动化

AI coding agent 可以快速补测试，但测试质量需要人类审查：

- 是否覆盖真实风险。
- 是否只断言实现细节。
- 是否引入脆弱 mock。
- 是否能在 CI 中稳定运行。

## 本章小结

让 agent 做调试、重构和测试，必须把反馈信号变成第一等公民。没有测试的自动重构只是更快地产生风险。
