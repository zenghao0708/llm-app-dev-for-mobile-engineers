# 第 13 章 AI Coding 效果评测

## 本章导读

AI coding 是否有效，不能只靠主观感觉。团队需要评估任务成功率、修改质量、测试覆盖、审查成本和回归风险。

## 学习目标

- 建立 AI coding 评测指标。
- 知道如何设计移动端任务集。
- 能够区分效率提升和质量下降。

## 评测维度

| 维度 | 指标 |
| --- | --- |
| 任务完成 | 完成率、一次通过率、平均轮数 |
| 代码质量 | diff 大小、架构一致性、重复代码、复杂度 |
| 测试验证 | 测试是否新增、测试是否通过、覆盖风险点 |
| 审查成本 | 人工修改次数、审查意见数量 |
| 安全合规 | 是否触碰敏感文件、是否泄露秘密 |
| 移动端质量 | 崩溃率、启动时间、包体、权限、兼容性 |

## 任务集设计

移动端 AI coding 评测任务可以包括：

- 修复一个有堆栈的崩溃。
- 给 ViewModel 补测试。
- 拆分一个复杂 UI 组件。
- 迁移一个网络错误模型。
- 修复一个跨端平台差异。
- 生成一份发版检查文档。

## 本章小结

评测的目标不是证明 agent 很强，而是找出它适合哪些任务、不适合哪些任务，以及需要什么 harness 才能稳定发挥。
