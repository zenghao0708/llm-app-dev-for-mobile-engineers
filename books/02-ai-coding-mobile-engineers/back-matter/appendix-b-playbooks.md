# 附录 B 移动端 AI Coding Playbook

## Playbook 1：崩溃修复

输入：

- 崩溃堆栈。
- 复现步骤。
- 设备和系统版本。
- 最近相关提交。

流程：

1. 让 agent 先解释堆栈，不改代码。
2. 定位相关文件。
3. 写出两个以内的修复假设。
4. 补一个失败测试。
5. 做最小修复。
6. 运行测试并输出结果。

## Playbook 2：UI 组件拆分

输入：

- 目标页面。
- 当前组件文件。
- 设计约束。
- 状态管理方式。

流程：

1. 识别可拆分区域。
2. 保持外部 API 不变。
3. 每次只抽取一个子组件。
4. 补充 snapshot 或 widget test。
5. 检查无障碍、本地化和暗黑模式。

## Playbook 3：依赖升级

输入：

- 当前版本。
- 目标版本。
- changelog。
- 构建命令。

流程：

1. 让 agent 总结 breaking changes。
2. 生成迁移计划。
3. 修改依赖和必要调用点。
4. 运行构建和测试。
5. 输出风险和回滚方式。

## Playbook 4：权限变更审查

输入：

- 新增权限。
- 使用场景。
- 平台配置文件。
- 用户文案。

流程：

1. 检查是否真的需要权限。
2. 确认 iOS/Android 配置一致。
3. 检查隐私说明和弹窗文案。
4. 检查是否影响审核。
5. 标记必须人工确认。
