# 附录 B 移动端 AI Coding Playbook

## Playbook 1：崩溃修复

输入：

- 崩溃堆栈。
- 复现步骤。
- 设备和系统版本。
- 最近相关提交。

流程：

1. 让 agent 先解释堆栈，不改代码。
2. 定位相关文件。
3. 写出两个以内的修复假设。
4. 补一个失败测试。
5. 做最小修复。
6. 运行测试并输出结果。

输出模板：

```text
根因：
- ...

证据：
- 堆栈：
- 相关代码：
- 新增测试：

修改：
- ...

验证：
- 命令：
- 结果：

未验证：
- ...
```

禁止事项：

- 不根据堆栈直接大范围重构。
- 不修改无关模块。
- 不把真实用户日志贴入 prompt。
- 不删除异常处理来让测试通过。

## Playbook 2：UI 组件拆分

输入：

- 目标页面。
- 当前组件文件。
- 设计约束。
- 状态管理方式。

流程：

1. 识别可拆分区域。
2. 保持外部 API 不变。
3. 每次只抽取一个子组件。
4. 补充 snapshot 或 widget test。
5. 检查无障碍、本地化和暗黑模式。

验收：

- 外部调用方式不变。
- UI 状态不变。
- 测试或截图覆盖 loading、empty、error、success。
- 本地化 key 没有丢失。
- 暗黑模式和动态字体没有明显问题。

适用范围：

- SwiftUI View 拆分。
- Jetpack Compose Composable 拆分。
- Flutter Widget 拆分。
- React Native Component 拆分。

## Playbook 3：依赖升级

输入：

- 当前版本。
- 目标版本。
- changelog。
- 构建命令。

流程：

1. 让 agent 总结 breaking changes。
2. 生成迁移计划。
3. 修改依赖和必要调用点。
4. 运行构建和测试。
5. 输出风险和回滚方式。

人工确认点：

- 是否必须升级。
- 许可证是否可接受。
- 包体或方法数是否明显增加。
- 是否影响最低系统版本。
- 是否需要同步 CI 镜像或缓存。

禁止事项：

- 不自动升级 Gradle wrapper、AGP、Kotlin、Xcode project 关键配置，除非任务明确要求。
- 不删除 lockfile。
- 不把依赖升级和业务重构混在同一个 PR。

## Playbook 4：权限变更审查

输入：

- 新增权限。
- 使用场景。
- 平台配置文件。
- 用户文案。

流程：

1. 检查是否真的需要权限。
2. 确认 iOS/Android 配置一致。
3. 检查隐私说明和弹窗文案。
4. 检查是否影响审核。
5. 标记必须人工确认。

输出：

- 新增或修改的权限。
- 使用场景。
- 用户可见文案。
- iOS 配置位置。
- Android 配置位置。
- 替代方案。
- 必须人工验证的设备场景。

## Playbook 5：网络错误模型迁移

输入：

- 当前错误模型。
- 网络层封装。
- 2-3 个低风险页面。
- 测试命令。
- 本地化和埋点规则。

流程：

1. 只读分析错误处理分布。
2. 输出迁移表，按风险排序。
3. 第一轮只新增统一错误模型和 mapper。
4. 为 mapper 补测试。
5. 第二轮迁移低风险页面。
6. 人工确认文案和埋点。
7. 分批迁移剩余页面。

停止条件：

- 需要修改支付、登录、风控。
- 需要修改接口协议。
- 需要新增依赖。
- 测试连续失败两轮仍无法定位。

## Playbook 6：跨端平台通道排查

输入：

- Flutter MethodChannel 或 RN Native Module 名称。
- Dart/TypeScript 调用方。
- iOS 实现。
- Android 实现。
- 错误日志。

流程：

1. 对比跨端层参数和原生层参数。
2. 检查 iOS/Android 是否都有实现。
3. 检查错误码和异常映射是否一致。
4. 补充跨端层测试。
5. 若涉及权限，转入权限变更审查 playbook。

验收：

- 双平台协议一致。
- 用户取消、权限拒绝、系统不可用都有兜底。
- 未验证平台被明确列出。

## Playbook 7：AI Coding PR 审查

输入：

- PR diff。
- agent 输出的测试结果。
- 任务原始 prompt。
- 高风险文件清单。

流程：

1. 检查是否超出任务范围。
2. 检查是否修改高风险文件。
3. 检查测试是否覆盖风险点。
4. 检查是否新增依赖或配置变更。
5. 检查 PR 描述是否包含未验证项。
6. 给出合并、返工或人工验证建议。

审查结论模板：

```text
结论：可合并 / 需修改 / 暂停

主要问题：
- ...

必须补充：
- ...

可后续处理：
- ...
```
