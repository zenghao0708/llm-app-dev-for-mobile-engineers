# 附录 A 端侧 AI 术语表

| 术语 | 简要解释 | 移动端关注点 |
| --- | --- | --- |
| 端侧 AI | 在用户设备上运行的 AI 能力。 | 关注性能、电量、隐私和离线体验。 |
| Runtime | 模型执行环境。 | 影响模型格式、硬件加速和调试方式。 |
| 量化 | 降低模型数值精度以减少体积和计算量。 | 可能影响效果，需要评测。 |
| 端云协同 | 端侧和云端共同完成任务。 | 需要路由、降级、隐私边界。 |
| 本地检索 | 在设备上建立索引并召回内容。 | 要处理加密、删除和同步。 |
| 灰度 | 分批向用户开放能力。 | 需要指标和回滚。 |
| 能力查询 | App 在运行前查询当前设备是否支持某项 AI 能力。 | 避免盲目调用模型，支持降级和提示。 |
| 模型包 | 模型权重、tokenizer、配置、manifest 和校验信息的集合。 | 需要下载、校验、安装和回滚。 |
| Manifest | 描述模型包版本、大小、checksum、兼容性和 schema 的文件。 | 是客户端判断能否加载模型的依据。 |
| Checksum | 文件完整性校验值。 | 防止半包、损坏包或被篡改文件进入 runtime。 |
| 原子安装 | 下载和校验完成后一次性切换到新模型。 | 避免 App 被杀掉后留下半安装状态。 |
| Schema Version | 输入输出结构版本。 | 保障模型、索引和调用方兼容。 |
| Index Version | 本地索引结构版本。 | embedding 模型或分段规则变化后决定是否重建。 |
| KV Cache | 生成式模型推理过程中保存的中间状态。 | 影响内存峰值和长文本生成能力。 |
| Tokenizer | 把文本转换为模型 token 的组件。 | 必须和模型版本绑定，不能随意混用。 |
| Embedding | 把文本、图片或其他输入映射成向量。 | 可用于检索和个性化，但仍可能包含敏感信息。 |
| 量化回归集 | 用于比较量化前后效果变化的固定样本集。 | 要覆盖边界输入和高风险样本。 |
| 置信度 | 模型对结果可靠性的估计。 | 决定自动执行、用户确认或拒答。 |
| 拒答 | 模型或系统主动不输出结果。 | 用于低置信度、高风险和超出边界场景。 |
| 降级 | 在模型不可用或超预算时切换到较弱但可用的路径。 | 是移动端稳定性的核心设计。 |
| 端云路由 | 决定任务在本地、云端或端云协同中执行。 | 要考虑隐私、网络、能力和成本。 |
| 隐私过滤 | 上云前识别并处理敏感内容。 | 需要脱敏预览、用户确认和高风险拒绝。 |
| 脱敏预览 | 展示上传前被遮挡或替换后的内容。 | 提高用户信任，减少误上传。 |
| 本地索引 | 在设备上保存可检索结构。 | 原文删除、权限撤销和模型升级时要同步处理。 |
| 设备矩阵 | 发布前覆盖的机型、系统版本和设备能力组合。 | 低端设备结果通常决定发布边界。 |
| P95/P99 | 分位延迟指标。 | 比平均值更能反映真实用户等待体验。 |
| OOM | 内存不足导致进程被系统杀死或崩溃。 | 多见于大图、多模态和长文本场景。 |
| ANR | Android 应用无响应。 | AI 任务阻塞主线程或资源过载时可能出现。 |
| Bridge | 跨平台框架与原生层之间的通信通道。 | 大文件穿过 bridge 会造成明显性能问题。 |
| 能力 Bundle | 一组经过验证的模型、索引和配置组合。 | 用于保持多模型依赖的一致性。 |
