# 《端侧大模型与移动智能工程》书稿项目

本目录维护三部曲第三册书稿。第三册面向移动端工程师，主题是端侧大模型、混合推理架构、隐私保护、性能优化和移动智能产品工程。

## 定位

- 系列位置：三部曲第三册。
- 目标篇幅：约 10 万字，合理区间 9 万-11 万字。
- 目标读者：移动端架构师、端侧 AI 工程师、iOS/Android/Flutter/RN 工程师、AI 产品工程师。
- 前置基础：理解第一册的大模型应用开发基础，并掌握第二册的 AI coding 工程化方法。
- 核心问题：如何在移动端把大模型能力做成可用、可控、可评测、可发布的端侧或端云协同产品。

## 写作主线

第三册关注五条主线：

1. 端侧模型能力：小模型、语音、视觉、embedding、LLM runtime。
2. 端云协同：本地优先、云端兜底、隐私边界、成本控制。
3. 移动端性能：启动、内存、电量、包体、热管理、离线体验。
4. 产品工程：输入输出体验、失败降级、用户可控、A/B 与灰度。
5. 发布治理：模型更新、评测、监控、合规和长期维护。

## 当前状态

当前版本已形成约 10 万字符的完整初稿：包含书名页、前言、目录、14 章和附录 A-D，并可通过根目录构建脚本生成 Markdown 审校稿和 EPUB 电子书。

- 对端侧 runtime、系统 API、模型格式和隐私合规表述做事实核验。
- 为核心链路补充高清图表、截图或流程图，并检查 EPUB 阅读器渲染效果。
