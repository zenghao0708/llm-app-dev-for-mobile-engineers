# 第 2 章 移动端 AI Runtime 与模型格式

## 本章导读

端侧 AI 需要 runtime、模型格式和硬件加速共同工作。工程师不一定要训练模型，但必须理解模型如何被加载、执行、量化和更新。

## 学习目标

- 理解 Core ML、TensorFlow Lite、ONNX Runtime 等 runtime 的定位。
- 知道模型格式、量化和硬件后端对移动端体验的影响。
- 能为项目建立 runtime 选择清单。

## Runtime 选择维度

| 维度 | 关注点 |
| --- | --- |
| 平台 | iOS、Android、跨平台 |
| 模型类型 | 文本、语音、视觉、embedding、LLM |
| 性能 | 延迟、吞吐、冷启动、内存 |
| 更新 | 模型是否随包发布，是否远程下载 |
| 调试 | 是否有 profiling 和日志工具 |

## 工程边界

Runtime 选择不是纯技术偏好。它会影响包体、CI、崩溃定位、灰度策略和线上观测。团队应在选型阶段记录决策原因，而不是等到发版前才发现设备不兼容。

## 本章小结

模型格式和 runtime 是端侧 AI 的执行底座。选型时要同时考虑能力、性能、更新和团队维护成本。
