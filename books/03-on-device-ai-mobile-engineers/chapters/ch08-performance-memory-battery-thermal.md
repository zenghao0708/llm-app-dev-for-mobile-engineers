# 第 8 章 性能、内存、电量与热管理

## 本章导读

端侧模型进入 App 后，性能问题会直接影响用户体验。延迟、内存、电量和发热必须在开发阶段被量化，而不是发版后再观察。

## 学习目标

- 建立端侧 AI 性能指标。
- 设计本地 profiling 和线上观测。
- 理解低端设备和长时间运行风险。

## 指标清单

| 指标 | 说明 |
| --- | --- |
| 首次加载时间 | 模型首次可用所需时间 |
| 单次推理延迟 | 用户等待时间 |
| 峰值内存 | 是否触发系统回收或崩溃 |
| 电量消耗 | 长任务和后台任务影响 |
| 温度 | 是否触发降频 |
| 包体 | 模型是否显著增加安装成本 |

## 本章小结

端侧 AI 的性能优化必须从指标开始。没有指标的优化只是猜测。
