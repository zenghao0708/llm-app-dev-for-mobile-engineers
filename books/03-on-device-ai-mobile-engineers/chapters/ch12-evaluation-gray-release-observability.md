# 第 12 章 评测、灰度与线上观测

## 本章导读

端侧 AI 发布前需要离线评测，发布后需要灰度和观测。模型效果、性能、崩溃和用户体验都要进入指标体系。

## 学习目标

- 设计端侧 AI 评测任务集。
- 建立灰度、回滚和观测方案。
- 区分模型质量问题和移动端工程问题。

## 指标体系

| 维度 | 指标 |
| --- | --- |
| 效果 | 准确率、召回率、用户接受率 |
| 性能 | 延迟、内存、电量、温度 |
| 稳定性 | 崩溃、ANR、模型加载失败 |
| 体验 | 取消率、重试率、降级率 |
| 隐私 | 本地处理比例、上传数据类型 |

## 本章小结

端侧 AI 的发布不能只看模型效果。移动端工程指标和用户体验指标同样重要。
