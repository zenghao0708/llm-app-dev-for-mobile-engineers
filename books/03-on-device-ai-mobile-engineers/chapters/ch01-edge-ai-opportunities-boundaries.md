# 第 1 章 端侧大模型的机会与边界

## 本章导读

端侧大模型不是把云端模型缩小后放进手机。移动设备有严格的电量、内存、温度、网络、权限和发布约束。一个模型能在实验环境中运行，不代表能进入真实 App。

## 学习目标

- 理解端侧智能的收益和边界。
- 区分端侧推理、云端推理和端云协同。
- 知道哪些场景适合本地优先，哪些场景必须云端兜底。

## 核心问题

端侧 AI 的价值主要来自四点：低延迟、隐私保护、离线可用和成本控制。但它也带来模型包体、运行性能、设备兼容、评测和更新治理问题。

移动端工程师要避免两个极端：一个是所有任务都上云，忽略隐私和体验；另一个是所有任务都端侧化，忽略能力、成本和维护复杂度。

## 场景判断表

| 场景 | 适合端侧 | 适合云端 |
| --- | --- | --- |
| 输入法联想、简单分类 | 是 | 可选 |
| 离线语音命令 | 是 | 可选 |
| 大规模知识问答 | 通常不适合 | 是 |
| 隐私敏感本地摘要 | 是 | 需谨慎 |
| 高质量长文本生成 | 视设备而定 | 是 |

## 本章小结

端侧大模型的关键不是“能不能跑”，而是“是否值得跑、怎样稳定跑、失败时如何降级”。后续章节会从 runtime、架构、性能、隐私和产品化角度展开。
