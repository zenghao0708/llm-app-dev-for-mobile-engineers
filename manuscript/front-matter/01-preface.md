# 前言

大模型正在改变软件的构建方式。过去，软件系统主要依靠确定性的业务规则、数据库查询和接口调用来完成任务；今天，越来越多的系统开始把自然语言理解、文本生成、知识检索、工具调用和多步骤推理能力嵌入到业务流程中。

对于移动端开发工程师来说，学习大模型并不意味着必须从神经网络、反向传播和模型训练开始。更现实的路径是：先理解大模型能做什么、不能做什么，然后掌握如何通过 API、Prompt、RAG、Agent 和工程化方法把大模型接入真实 App。

本书采用入门与实战结合的方式组织内容。前几章帮助读者建立基础认知；中间章节围绕 Prompt、API、结构化输出、Embedding 和 RAG 展开；后续章节介绍 Agent、多工具调用、多模态输入、移动端交互和企业级工程实践；最后通过一个移动端知识助手项目串联全书知识。

本书中的完整代码以 Python 服务端示例为主，尽量使用通用 HTTP 请求和简单依赖，避免让读者一开始就陷入复杂框架。移动端部分会重点说明接口契约、流式渲染、取消请求、权限申请、文件上传、缓存和隐私处理。实际开发中，读者可以根据团队情况替换为具体厂商 SDK、企业内部模型网关或开源推理服务。

需要特别说明的是，大模型生态变化很快。模型名称、SDK 方法、平台控制台和移动端系统能力都可能随时间调整。因此，本书更关注稳定的工程模式：上下文组织、结构化约束、检索增强、工具编排、移动端状态管理、评测闭环、安全边界和成本治理。掌握这些方法后，即使底层模型、平台或客户端技术栈发生变化，读者也能较快迁移。
