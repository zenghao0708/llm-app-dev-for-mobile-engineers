# 每章高清图片生成与截图提示

以下提示用于制作配套高清图片。原理图建议用 SVG 或 Figma 重绘，截图建议来自真实运行项目。

## 通用视觉要求

白底或浅灰底，深蓝、青绿和石墨灰为主色。图中文字少而清楚，箭头和模块对齐，避免炫光、赛博朋克和复杂背景。每张图至少 3000 px 宽，或使用 SVG 矢量格式。

## 第 1 章

图片提示：一张简洁的移动端大模型应用调用流程图，包含 User、Mobile App、Server API、Prompt Builder、LLM API、Response Parser、Business System 七个模块，使用专业流程图连接线，箭头尖端刚好与模块边框交汇，白底，技术教材风。

## 第 2 章

图片提示：展示上下文窗口的横向长条图，分为 System Prompt、User Input、Retrieved Context、Model Output 四段，旁边标注 Token Budget，风格简洁。

## 第 3 章

截图建议：终端中展示 Python 版本、虚拟环境创建、环境变量设置和一次服务端 API 调用输出。隐藏 API Key，只显示 `LLM_API_KEY=***`。补充移动端网络调试截图或模拟器请求截图。

## 第 4 章

图片提示：Prompt 结构拆解图，包含角色、任务、上下文、约束、输出格式、示例六个层次，使用模块化卡片表示。

## 第 5 章

图片提示：同步 API 调用和流式 API 调用对比图。同步调用为一个完整响应，流式调用为多个 token chunk 连续返回。

## 第 6 章

图片提示：工具调用链路图，模型根据用户问题选择工具，服务端执行工具并把结果返回模型，最后生成自然语言回答。移动端只展示确认和结果。

## 第 7 章

图片提示：Embedding 映射图，左侧为文本片段，右侧为二维向量空间中的点，距离近表示语义相似。

## 第 8 章

图片提示：RAG 管线图，包含文档导入、切分、Embedding、向量库、查询、召回、生成、引用来源。

## 第 9 章

图片提示：三种 Chunk 策略对比图：固定长度、按标题切分、语义切分，并用不同颜色展示边界。

## 第 10 章

图片提示：Agent 循环图，包含 Observe、Plan、Act、Tool Result、Reflect 五个节点，循环箭头连接。

## 第 11 章

图片提示：工作流编排图，展示确定性流程、模型决策节点、工具调用节点和人工确认节点之间的关系。

## 第 12 章

图片提示：多模态输入图，展示文本、截图、相册图片、语音和 PDF 经过移动端权限与服务端处理后进入模型服务，再输出结构化结果和自然语言说明。

## 第 13 章

图片提示：大模型评测闭环图，包含测试集、模型输出、自动评分、人工复核、Prompt/检索策略迭代。

## 第 14 章

图片提示：成本与性能监控仪表盘示意图，包含 Token 用量、平均延迟、错误率、缓存命中率四个指标。

## 第 15 章

图片提示：安全边界图，展示用户输入经过 Prompt Injection 检测、权限检查、工具白名单和日志脱敏后进入模型。

## 第 16 章

图片提示：移动端知识助手完整架构图，包含移动端 App、服务端 API、任务队列、文档解析、向量库、模型网关、评测系统和监控系统。连接线采用正交折线或专业流程图连接点，箭头尖端刚好停在模块边框。
